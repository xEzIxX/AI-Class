# 텍스트 분류 딥러닝 실습

## 학습 목적

이번 실습은 텍스트 데이터를 딥러닝 모델에 넣기 위해 어떻게 전처리하고,
Embedding을 사용해 문장을 분류하는지 이해하는 것

* 텍스트는 그대로 모델에 넣을 수 없기 때문에 숫자로 바꾸고,
문장 길이를 맞춘 뒤 신경망 모델에 입력한다.

---

## 1. IMDB 영화 리뷰 감성 분석

IMDB 영화 리뷰 데이터를 사용해서 리뷰가 긍정인지 부정인지 분류하는 실습이다.

```text
1 = 긍정 리뷰
0 = 부정 리뷰
```

이 실습에서는 영화 리뷰 문장을 숫자로 변환하고,
Embedding Layer를 통해 단어를 벡터로 바꾼 뒤 긍정/부정을 예측했다.

### 배운 내용

* 영화 리뷰 같은 텍스트 데이터도 딥러닝으로 분류할 수 있다.
* 문장은 모델에 넣기 전에 숫자로 변환해야 한다.
* 리뷰마다 길이가 다르기 때문에 `pad_sequences()`로 길이를 맞춰야 한다.
* `Embedding`은 단어를 벡터 형태로 바꿔주는 역할을 한다.
* 출력값이 0.5보다 크면 긍정, 작으면 부정으로 판단할 수 있다.

---

## 2. 스팸 메일 분류

짧은 문장 데이터를 사용해서 스팸 메일인지 정상 메일인지 분류하는 실습이다.

```text
1 = 스팸 메일
0 = 정상 메일
```

간단한 이메일 문장들을 직접 만들고,
각 문장을 숫자로 바꾼 뒤 Embedding 기반 모델로 스팸 여부를 예측했다.

### 배운 내용

* 간단한 문장 데이터도 직접 만들어서 학습시킬 수 있다.
* `one_hot()`을 사용하면 문장을 숫자 리스트로 바꿀 수 있다.
* 문장 길이가 다르면 패딩을 통해 같은 길이로 맞춰야 한다.
* Embedding Layer를 사용하면 단어 번호를 학습 가능한 벡터로 바꿀 수 있다.
* `sigmoid` 출력값을 기준으로 스팸/정상을 판단할 수 있다.

---

## 전체 요약 정리

- 텍스트 분류 흐름

```text
텍스트 데이터
→ 정수 인코딩
→ 패딩
→ Embedding
→ 신경망 학습
→ 분류 결과 예측
```

- 딥러닝 모델은 문자를 직접 이해하지 못하기 때문에, 텍스트를 숫자로 바꾸는 과정이 먼저 필요
- 문장마다 길이가 다르기 때문에 같은 길이로 맞추는 패딩 과정이 필요하다.
- Embedding은 단어를 단순한 숫자가 아니라 의미를 가진 벡터처럼 다룰 수 있게 해준다.
