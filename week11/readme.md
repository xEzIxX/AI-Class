## 오늘 배운 내용 정리 — RNN 중심

### 1. Dense Layer / CNN / RNN의 차이

| 구분              | 핵심 특징                        | 예시                       |
| --------------- | ---------------------------- | ------------------------ |
| **Dense Layer** | 모든 입력을 일렬로 펼쳐서 연결함           | 이미지의 픽셀 위치 관계를 잘 반영하지 못함 |
| **CNN**         | 현재 입력의 **공간적 특징**을 추출함       | 이미지에서 선, 모서리, 패턴 찾기      |
| **RNN**         | 현재 입력뿐 아니라 **이전 정보**도 함께 반영함 | 주식, 문장, 음악, 음성 같은 순차 데이터 |

```text
Dense Layer: 입력 전체를 단순 연결
CNN: 현재 입력의 공간적 특징 반영
RNN: 현재 입력 + 이전 정보 반영
```

---

## 2. CNN과 RNN의 가장 큰 차이

### CNN

CNN은 주로 **현재 입력의 특징**을 본다.

예를 들어 이미지 분류에서는 현재 이미지 안에 있는 픽셀들의 관계를 보고 특징을 추출한다.

```text
현재 이미지 → CNN → 결과
```

즉, CNN은 공간적인 관계를 잘 본다.

```text
이미지의 픽셀 간 인접 관계
모서리
선
패턴
형태
```

---

### RNN

RNN은 **이전 정보(previous information)** 를 반영한다.

```text
이전 정보 + 현재 입력 → RNN → 결과
```

그래서 RNN은 순서가 중요한 데이터에 사용된다.

예시:

```text
주식
음악
음성
문장
날씨
생산량
```

즉, RNN의 핵심은 **과거 정보를 기억하면서 현재 값을 예측하는 것**이다.

---

## 3. RNN의 핵심 개념

### RNN은 이전 정보를 Hidden State에 저장한다

RNN은 매 시점마다 입력을 받고, 그 정보를 **은닉 상태(hidden state)** 에 저장한다.

```text
현재 입력 x_t
이전 은닉 상태 h_t-1
        ↓
새로운 은닉 상태 h_t
```

수식으로 보면:

```text
h_t = f(x_t, h_t-1)
```

의미는:

```text
새로운 은닉 상태 = 현재 입력 + 이전 은닉 상태를 반영한 결과
```

즉, RNN은 단순히 현재 입력만 보는 것이 아니라, 이전 시점의 정보를 계속 넘겨받는다.

---

## 4. RNN이 다루는 데이터: 순차 데이터

RNN은 **순서가 있는 데이터**를 다룬다.

이런 데이터를 **순차 데이터** 또는 **시계열 데이터**라고 한다.

예시:

```text
주식 가격
음성 데이터
음악 데이터
문장 데이터
날씨 데이터
생산량 데이터
```

이 데이터들의 공통점은 **이전 값이 현재 값에 영향을 준다**는 것이다.

예를 들어 주식에서는 오늘 가격만 보는 것이 아니라, 최근 며칠간의 흐름을 함께 봐야 한다.

---

## 5. Window Size 개념

RNN에서 중요한 개념이 **window size**이다.

window size는 모델이 한 번에 참고할 과거 데이터의 길이를 의미한다.

예를 들어 window size가 3이면:

```text
1일차, 2일차, 3일차 데이터 → 4일차 예측
2일차, 3일차, 4일차 데이터 → 5일차 예측
3일차, 4일차, 5일차 데이터 → 6일차 예측
```

즉, 윈도우를 한 칸씩 밀어가면서 학습 데이터를 만든다.

---

## 6. Many-to-One 구조

이번에 배운 주식 예측 모델은 **Many-to-One** 구조이다.

```text
여러 개의 입력 → 하나의 출력
```

예시:

```text
3일치 Open 값 → 다음 날 Close 하나 예측
```

구조로 보면:

```text
Open1
Open2
Open3
  ↓
Close4 예측
```

즉, 여러 시점의 데이터를 보고 하나의 값을 예측한다.

---

## 7. Embedding과 Representation

RNN을 포함한 딥러닝 모델은 데이터를 그대로 이해하는 것이 아니라, 데이터를 숫자 벡터로 바꿔서 이해한다.

이때 사용하는 개념이 **embedding**이다.

예를 들어:

```text
남자 → 숫자 벡터
여자 → 숫자 벡터
킹 → 숫자 벡터
퀸 → 숫자 벡터
```

모델은 비슷한 의미를 가진 데이터가 임베딩 공간에서 가까이 위치하도록 학습한다.

예를 들어:

```text
킹은 남자와 가까운 위치
퀸은 여자와 가까운 위치
```

이처럼 무질서한 데이터를 의미 있는 벡터 공간에 표현하는 것을 **representation**이라고 한다.

---

## 8. Encoder와 Decoder

### Encoder

Encoder는 정보를 압축해서 벡터로 만드는 역할을 한다.

```text
입력 데이터 → 압축된 벡터
```

즉, 중요한 정보를 요약해서 표현한다.

이 압축된 공간을 **latent space**라고 한다.

---

### Decoder

Decoder는 압축된 정보를 다시 복원하거나 확장하는 역할을 한다.

```text
압축된 벡터 → 출력 데이터
```

정리하면:

```text
Encoder: 정보를 압축
Decoder: 압축된 정보를 다시 확장
```

---

## 8. RNN의 한계

RNN은 이전 정보를 반영할 수 있지만, 입력 길이가 길어지면 문제가 생긴다.

대표적인 문제가 두 가지이다.

```text
Gradient Vanishing
Gradient Exploding
```

### Gradient Vanishing

기울기가 점점 작아져서 앞쪽 정보가 거의 학습되지 않는 문제이다.

```text
긴 문장
긴 시계열 데이터
오래된 과거 정보
```

를 잘 기억하지 못할 수 있다.

---

### Gradient Exploding

기울기가 너무 커져서 학습이 불안정해지는 문제이다.

즉, RNN은 긴 순서 데이터를 다룰 때 한계가 있다.

---

## 9. LSTM이 등장한 이유

RNN은 긴 데이터를 학습할 때 과거 정보를 잘 잊어버리는 문제가 있다.

이 문제를 완화하기 위해 나온 모델이 **LSTM**이다.

LSTM은 중요한 정보는 오래 기억하고, 필요 없는 정보는 버리는 구조를 가진다.

즉:

```text
RNN: 이전 정보를 단순히 넘김
LSTM: 중요한 정보는 기억하고, 덜 중요한 정보는 제거
```

그래서 긴 문장 번역이나 긴 시계열 데이터 처리에서 RNN보다 LSTM이 더 유리하다.
