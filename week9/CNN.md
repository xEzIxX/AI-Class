## CNN

CNN은 이미지의 공간적인 특징을 유지하면서 학습하기 위해 사용되는 딥러닝 모델이다.  
일반적인 Dense Network 또는 Fully Connected Network는 이미지를 1차원으로 평탄화하여 처리하지만, CNN은 Convolution 연산과 Pooling을 통해 이미지의 특징을 효과적으로 추출한다.

> ### 교수님의 CNN 이해 정리
>
> 1. 이미지의 공간적인 특징을 평탄화(dense / fully connected network)하게 되면, 공간적인 특성을 capture하지 못한다.
> 2. 이런 단점을 보완하기 위해서 Convolution (합성곱)연산을 한다.
> 3. 합성곱은 원본에 새로운 필터를 곱해서 새로운 특징을 만들어내는 것이다.
> 4. 필터크기에 따라서 생성되는 가중치/바이어스는 더 증가하게 된다. (Dense 는 단 하나의 가중치/바이어스, CNN은 다수의 가중치/바이어스)
> 5. 합성곱(convolution)연산을 하게 되면 정보의 양이 급증
> 6. 풀링을 통해 정보의 크기를 줄인다 
> 7. 풀링은 최대풀링/평균풀링이 있다.
> 8. 풀링은 정보요약 / 이동에 대한 불변 2가지 기능이 있다.
> 9. Stride는 움직이는 거리, Padding는 경계처리 여부에 관한 것이다.
> 10. 필터는 학습을 통해서 최적화 된다. (초기에는 랜덤값)

## 궁금점

### Q1. 왜 평탄화되면 공간적인 특성을 capture하지 못하는가?

이미지는 가로와 세로의 위치 관계가 중요한 데이터이다.  
예를 들어 어떤 픽셀이 다른 픽셀의 옆에 있는지, 위아래에 있는지가 이미지의 의미를 결정한다.

하지만 이미지를 평탄화하면 2차원 이미지가 1차원 벡터로 바뀐다.

예를 들어 3×3 이미지는 원래 다음과 같은 구조를 가진다.

```text
1 2 3
4 5 6
7 8 9
```

이를 평탄화하면 다음과 같이 바뀐다.

```text
1 2 3 4 5 6 7 8 9
```

이렇게 되면 **원래 이미지에서 가까이 있던 픽셀 간의 위치 관계가 약해진다** 
즉, 이미지의 **공간적인 구조**를 충분히 **반영하기 어렵다.**

따라서 Dense Network는 이미지의 공간적 특징을 직접적으로 capture하기 어렵다.

---

### Q2. 합성곱은 어떻게 동작하는가?

합성곱, 즉 Convolution은 **작은 Filter를 이미지 위에 이동시키면서 특징을 추출**하는 연산이다.

Filter는 작은 행렬 형태를 가진다.  
예를 들어 3×3 Filter는 다음과 같다.

```text
1 0 -1
1 0 -1
1 0 -1
```

이 Filter를 이미지의 일부 영역에 겹친 뒤, 같은 위치끼리 곱하고 모두 더한다.  
그 결과 하나의 숫자가 만들어진다.

이 과정을 이미지 전체에 반복하면 새로운 특징 정보인 Feature Map이 만들어진다.

즉, Convolution(합성곱)은 다음과 같이 이해할 수 있다.

> **Filter를 이미지 위에 움직이며, 특정 특징이 있는지 검사하는 과정**

Filter는 **처음에는 Random Value로 시작**하지만, **학습을 통해** 점점 이미지 분류에 유용한 값으로 **최적화**된다.

### Q3. Pooling은 어떻게 정보 크기를 줄이는가?

Pooling은 **Feature Map의 일정 영역을 하나의 값으로 요약**하여 크기를 줄이는 과정이다.

예를 들어 2×2 영역이 있다고 하자.

```text
1 3
2 4
```

Max Pooling을 하면 가장 큰 값인 4만 남긴다.

```text
4
```

Average Pooling을 하면 **평균값을 계산**한다.

```text
(1 + 3 + 2 + 4) / 4 = 2.5
```

즉, Pooling은 **여러 개의 값을 하나의 대표값으**로 줄이기 때문에 Feature Map의 **크기가 작아진다.**  
이를 통해 **계산량을 줄이고, 중요한 정보만 요약**할 수 있다.

### Q4. Max Pooling / Average Pooling의 특징

#### Max Pooling

Max Pooling은 정해진 영역에서 가장 큰 값을 선택하는 방식이다.

```text
1 3
2 4
```

결과:

```text
4
```

**가장 강하게 반응한 특징**을 남기기 때문에, **이미지에서 뚜렷한 특징을 보존하**는 데 유리하다.  
예를 들어 선, 모서리, 특정 패턴이 강하게 나타난 부분을 잘 살릴 수 있다.

#### Average Pooling

Average Pooling은 **정해진 영역의 평균값을 계산**하는 방식이다.

```text
1 3
2 4
```

결과:

```text
2.5
```

전체적인 정보를 **부드럽게 요약**하는 데 유리하다.  
하지만 **강한 특징**이 평균에 의해 **약해질 수 있다.**

| 구분 | Max Pooling | Average Pooling |
|---|---|---|
| 방식 | 가장 큰 값 선택 | 평균값 계산 |
| 특징 | 강한 특징 보존 | 전체 정보 요약 |
| 장점 | 선, 모서리 등 뚜렷한 특징 유지 | 전체적인 흐름 반영 |
| 단점 | 작은 정보는 버려질 수 있음 | 강한 특징이 약해질 수 있음 |

### Q5. Pooling은 어떻게 정보 요약과 이동에 대한 불변성을 제공하는가?

Pooling은 **일정 영역 안의 여러 값**을 **하나의 대표값으로** 줄인다.  
따라서 전체 정보를 세세하게 모두 저장하지 않고, **중요한 정보만 요약**한다.

예를 들어 Max Pooling은 2×2 영역 안에서 **가장 큰 값만 남긴다.**

```text
1 8
2 3
```

결과:

```text
8
```

이 경우 정확히 **어느 위치에** 8이 있었는지는 **중요하지 않고,**  
“**이 영역 안에** 강한 특징이 **있었다**”는 정보만 남는다.

이 때문에 이미지 속 특징이 조금 이동해도 비슷한 결과가 나올 수 있다.

예를 들어 다음 두 영역은 8의 위치가 다르다.

```text
1 8
2 3
```

```text
8 1
2 3
```

하지만 Max Pooling 결과는 둘 다 8이다.

즉, Pooling은 특징의 정확한 위치보다는 특징의 존재 여부를 남기기 때문에,  
**이미지 안에서 물체가 조금 이동해도 비슷하게 인식**할 수 있도록 도와준다.

이것을 **이동에 대한 불변성**이라고 한다.

### Q6. Stride와 Padding은 어디서 등장하는 용어인가?

Stride와 Padding은 Convolution 연산이나 Pooling 연산에서 등장하는 용어이다.

| 용어 | 의미 | 효과 |
|---|---|---|
| Stride | Filter가 이동하는 간격 | 출력 크기 조절 |
| Padding | 이미지 가장자리에 값을 추가하는 방식 | 경계 정보 보존, 출력 크기 조절 |

#### Stride

Stride는 Filter 또는 **Pooling Window가 한 번에 이동하는 간격**을 의미한다.

Stride가 1이면 한 칸씩 이동한다.

```text
한 칸씩 촘촘하게 이동
```

Stride가 2이면 두 칸씩 이동한다.

```text
두 칸씩 건너뛰며 이동
```

**Stride가 커질수록 출력 Feature Map의 크기는 작아진다.**

#### Padding

Padding은 **이미지의 가장자리에 0과 같은 값을 추가하는 방식**이다.

원본 이미지가 다음과 같다고 하자.

```text
1 2 3
4 5 6
7 8 9
```

Padding을 추가하면 다음과 같이 된다.

```text
0 0 0 0 0
0 1 2 3 0
0 4 5 6 0
0 7 8 9 0
0 0 0 0 0
```

Padding을 사용하는 이유는 **이미지의 가장자리 정보가 손실되는 것을 줄이기 위해서**이다.  
Padding이 없으면 Filter가 이미지 가장자리를 충분히 반영하지 못하고, 출력 크기도 계속 작아진다.
